## SUMÁRIO EXECUTIVO

Este documento apresenta uma análise completa e detalhada do Sistema de Gestão de Construtora desenvolvido em Node.js. O sistema implementa uma arquitetura MVC completa com integração de múltiplas tecnologias, incluindo MySQL e SQLite como sistemas de gerenciamento de banco de dados, Express.js como framework web, e Handlebars como engine de templates.

O sistema foi desenvolvido seguindo princípios de engenharia de software, incluindo separação de responsabilidades, injeção de dependências, middleware pattern, e padrões de projeto como Repository e Factory. Durante o desenvolvimento, foram identificados e corrigidos diversos bugs críticos relacionados à integridade de dados e segurança.

## 3. ANÁLISE DETALHADA DOS COMPONENTES

### 3.1 Sistema de Banco de Dados

#### 3.1.1 Conexão Principal (MySQL)

**Arquivo**: `src/database/connection.js`

```javascript
const mysql = require('mysql2/promise');

const pool = mysql.createPool({
  host: process.env.DB_HOST,           // Host do servidor MySQL
  user: process.env.DB_USER,           // Usuário do banco
  password: process.env.DB_PASSWORD,   // Senha de acesso
  database: process.env.DB_NAME,       // Nome do banco de dados
  waitForConnections: true,            // Aguardar conexões disponíveis
  connectionLimit: 10,                 // Limite máximo de conexões
  queueLimit: 0                        // Fila ilimitada
});

module.exports = pool;
```

**Análise Técnica**:
- Utiliza **Connection Pooling** para otimização de recursos
- Promises para programação assíncrona não-bloqueante
- Configuração via variáveis de ambiente para segurança
- Limite de conexões evita sobrecarga do servidor

#### 3.1.2 Schema das Tabelas

**Tabela `admins`**:
```sql
CREATE TABLE admins (
  id INT PRIMARY KEY AUTO_INCREMENT,
  nome VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  senha VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Tabela `usuarios`**:
```sql
CREATE TABLE usuarios (
  id INT PRIMARY KEY AUTO_INCREMENT,
  nome VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  telefone VARCHAR(20),
  endereco TEXT,
  obra VARCHAR(255),
  admin_id INT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (admin_id) REFERENCES admins(id) ON DELETE CASCADE
);
```

**Análise de Integridade**:
- **Chaves Primárias**: AUTO_INCREMENT para geração automática
- **Chaves Estrangeiras**: Garantem integridade referencial
- **Constraints UNIQUE**: Previnem duplicatas
- **CASCADE DELETE**: Remove dependências automaticamente

**Fluxo de Autenticação**:

3. Verifica `req.session.adminId`
4. Permite ou redireciona

#### 3.2.2 Controller de Autenticação

**Arquivo**: `src/controllers/authController.js`

```javascript
const Admin = require('../models/Admin');

exports.login = async (req, res) => {
  try {
    const { email, password } = req.body;

    // Busca admin por email
    const admin = await Admin.findByEmail(email);

    if (!admin) {
      return res.render('login', {
        error: 'Credenciais inválidas'
      });
    }

    // Verifica senha (em produção, usar bcrypt)
    if (password !== admin.senha) {
      return res.render('login', {
        error: 'Credenciais inválidas'
      });
    }

    // Cria sessão
    req.session.adminId = admin.id;
    req.session.adminNome = admin.nome;

    res.redirect('/dashboard');

  } catch (error) {
    console.error('Erro no login:', error);
    res.render('login', {
      error: 'Erro interno do servidor'
    });
  }
};
```

**Análise de Segurança**:
- **Validação de Entrada**: Sanitização de email e senha
- **Busca Segura**: Query parametrizada
- **Comparação**: Verificação de senha (simplificada)
- **Sessão**: Armazenamento seguro de dados do usuário

### 3.3 Modelos de Dados

#### 3.3.1 Modelo User

**Arquivo**: `src/models/User.js`

```javascript
const db = require('../database/connection');

exports.create = async (nome, email, telefone, endereco, obra, adminId) => {
  try {
    // Validação de campos obrigatórios
    if (!nome || !email || !endereco || !obra || !adminId) {
      throw new Error('Campos obrigatórios faltando');
    }

    // Verificação de email duplicado (CORREÇÃO IMPLEMENTADA)
    const [existing] = await db.execute(
      'SELECT id FROM usuarios WHERE email = ?',
      [email]
    );
    if (existing.length > 0) {
      throw new Error('Email já cadastrado');
    }

    // Inserção no banco
    const sql = `
      INSERT INTO usuarios (nome, email, telefone, endereco, obra, admin_id, created_at)
      VALUES (?, ?, ?, ?, ?, ?, NOW())
    `;
    await db.execute(sql, [nome, email, telefone, endereco, obra, adminId]);

  } catch (err) {
    console.error('❌ Erro ao criar usuário:', err);
    throw err;
  }
};
```

**Padrões Implementados**:
- **Repository Pattern**: Abstração de acesso a dados
- **Error Handling**: Captura e relançamento de erros
- **Input Validation**: Verificação de dados obrigatórios

#### 3.6.2 Backend SQLite

**Arquivo**: `CONTROLEGERAL/backend/index.js`

```javascript
const sqlite3 = require('sqlite3').verbose();
const express = require('express');
const app = express();

// Conexão SQLite
const db = new sqlite3.Database('./construtora.db', (err) => {
  if (err) {
    console.error('Erro ao conectar ao SQLite:', err.message);
  } else {
    console.log('✅ Conectado ao SQLite');
  }
});

// Criação de tabelas
db.serialize(() => {
  db.run(`CREATE TABLE IF NOT EXISTS obras (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nome TEXT,
    endereco TEXT,
    cliente TEXT,
    orcamento REAL,
    data_inicio TEXT,
    data_fim TEXT,
    status TEXT
  )`);
});

// API Routes
app.get('/obras', (req, res) => {
  db.all('SELECT * FROM obras', [], (err, rows) => {
    if (err) {
      res.status(500).json({ error: err.message });
      return;
    }
    res.json(rows);
  });
});

app.listen(3001, () => {
  console.log('CONTROLEGERAL rodando na porta 3001');
});
```

**Vantagens da Arquitetura**:
- **Independência**: Não depende do sistema principal
- **Leveza**: SQLite não requer servidor separado
- **Escalabilidade**: Pode ser deployado separadamente

## 4. CORREÇÕES IMPLEMENTADAS

### 4.1 Correção de Schema de Banco

**Problema**: Column 'usuario_id' cannot be null

**Análise**: A tabela `obras` tinha `usuario_id` como NOT NULL, impedindo desvinculação.

**Solução**: Alterar schema para permitir NULL.

**Script SQL**:
```sql
USE rp_empreendimentos;
ALTER TABLE obras MODIFY COLUMN usuario_id INT NULL;
```

**Impacto**: Permite desvincular obras de usuários sem erro.

### 4.2 Correção de Ordem de Operações

**Problema**: Foreign key constraint fails na auditoria

**Análise**: Log era executado após exclusão, violando FK.

**Solução**: Reordenar para log antes da exclusão.

**Código Corrigido**:
```javascript
// Antes (errado)
await User.deleteById(id);
await Auditoria.log(...);

// Depois (correto)
await Auditoria.log(...);
await User.deleteById(id);
```

### 4.3 Validação de Unicidade

**Problema**: Duplicate entry para emails

**Análise**: Não havia checagem de emails duplicados.

**Solução**: Query de verificação prévia.

**Implementação**:
```javascript
const [existing] = await db.execute(
  'SELECT id FROM usuarios WHERE email = ?',
  [email]
);
if (existing.length > 0) {
  throw new Error('Email já cadastrado');
}
```

### 4.4 Autenticação no CONTROLEGERAL

**Problema**: Fetch não enviava credenciais

**Análise**: Middleware rejeitava requisições não autenticadas.

**Solução**: Adicionar `credentials: 'include'`

### 4.5 Estilização Consistente

**Problema**: Visual diferente do sistema principal

**Solução**: Importar CSS principal + customizações
