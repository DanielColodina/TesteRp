# DOCUMENTAÇÃO COMPLETA DO SISTEMA DE GESTÃO DE CONSTRUTORA

## VISÃO GERAL DO SISTEMA

Este é um sistema web completo para gestão de construtora, desenvolvido em Node.js com Express.js, utilizando MySQL como banco de dados e Handlebars para renderização de templates. O sistema permite que administradores gerenciem usuários, obras, checklists de progresso, auditoria, e inclui um módulo adicional CONTROLEGERAL para gestão independente de obras, estoque, equipes e financeiro.

### ARQUITETURA GERAL

O sistema segue uma arquitetura MVC (Model-View-Controller) com as seguintes camadas:

1. **Apresentação (Views)**: Templates Handlebars com layout responsivo
2. **Controle (Controllers)**: Lógica de negócio e manipulação de requisições
3. **Modelo (Models)**: Interação com banco de dados MySQL
4. **Middleware**: Autenticação, validação, rate limiting
5. **Rotas**: Definição dos endpoints da API

## BANCO DE DADOS

### Conexão Principal (MySQL)

**Arquivo**: `src/database/connection.js`

```javascript
const mysql = require('mysql2/promise');
const pool = mysql.createPool({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});
```

**Configuração**: Utiliza pool de conexões para melhor performance.

### Tabelas Principais

1. **admins**: Armazena administradores do sistema
2. **usuarios**: Usuários/clientes vinculados a administradores
3. **obras**: Projetos de construção
4. **checklist**: Itens de progresso das obras
5. **auditoria**: Log de todas as ações no sistema
6. **historico**: Histórico detalhado de mudanças

### CONTROLEGERAL Database (SQLite)

Banco SQLite separado para o módulo CONTROLEGERAL com tabelas:

## AUTENTICAÇÃO E AUTORIZAÇÃO

### Middleware de Autenticação

**Arquivo**: `src/middlewares/isAuth.js`

```javascript
module.exports = (req, res, next) => {
  if (req.session.adminId) {
    return next();
  }
  res.redirect('/login');
};
```

**Funcionalidade**: Verifica se existe adminId na sessão, redireciona para login se não autenticado.

### Sistema de Sessões

**Configuração**: Em `app.js`, utiliza express-session com armazenamento em memória, cookies seguros em produção.

### Rate Limiting

Protege endpoints de login contra ataques de força bruta (5 tentativas por 15 minutos).

## MODELOS (MODELS)

### User Model (`src/models/User.js`)

**Principais métodos**:
- `create()`: Insere novo usuário com validação de email duplicado
- `findAllByAdmin()`: Lista usuários de um admin
- `findAllWithAdmin()`: Lista todos com dados do admin

### Admin Model (`src/models/Admin.js`)

**Métodos**:
- `findByEmail()`: Busca admin por email
- `create()`: Cria novo admin

### Obra Model (`src/models/Obra.js`)

**Métodos**:
- `create()`: Cria nova obra
- `findById()`: Busca obra por ID
- `update()`: Atualiza obra
- `delete()`: Remove obra

### Checklist Model (`src/models/Checklist.js`)

**Funcionalidades**:
- Gerenciamento de checklist por usuário
- Cálculo de progresso
- Validação de campos obrigatórios

### Auditoria Model (`src/models/Auditoria.js`)

### Inicialização

1. Instalar dependências: `npm install`
2. Configurar banco: `node scripts/create_database_only.js`
3. Executar: `npm start`
